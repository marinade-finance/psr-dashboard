# 2026-05-15 — UX walkthrough as a 13-year-old, and two new skills

## What we did

Ran a full UX walkthrough of the PSR Dashboard with a fresh-novice
persona — a bright 13-year-old who can read the manual — and built
the methodology + reusable tooling on the same day.

End-to-end pipeline:

1. Captured the user's verbatim request in `docs/ux-13yo/instructions.md`
   so the brief survives context compression.
2. Spawned a research subagent on **frontend UX testing best practices**,
   targeting solo-evaluator + persona-driven methods. Output:
   `docs/ux-13yo/research-uxtest.md`.
3. Sent the research doc to `oracle` (codex CLI) with adversarial
   framing — got back a brutal critique. Oracle caught real factual
   errors:
   - Bailey & Wolfson 87/46 % was overclaimed (only applies to live
     task flows, not solo screenshot tests)
   - Hertzum & Jacobsen "22 %" was a misread — the range is 5–65 %
   - Cockton & Woolrych ">50 %" was overclaimed — the cited study
     reports 65 % in one specific setup
   - The 13yo persona is blind to financial judgement on a finance
     dashboard — the methodology was missing the most consequential
     dimension
4. Folded the critique into the methodology. Saved trail at
   `docs/ux-13yo/oracle-critique.md`.
5. Wrote `eye-13y` skill (UX walkthrough, 13yo lens) and `create`
   skill (research → oracle → mint a SKILL.md). Both at
   `~/.claude/skills/`.
6. Dogfooded `eye-13y` on the dashboard: 11 screens (SAM basic/expert,
   Bonds basic/expert, Events basic/expert, Docs basic/expert,
   validator detail, simulation, navigation).

## What we found in the dashboard

- **2 blockers, 93 frictions** across 11 reports.
- **Headline blocker:** validator-detail panel shows "Reserve: Fully
  covered" (green) next to "Bond below minimum — 7 SOL required"
  (orange) on the same validator. `GUIDE.md` explains the
  minimum-vs-ideal distinction, but the panel doesn't. Persona can't
  reconcile.
- **Headline delight:** simulation mode's banner —
  "SIMULATION MODE — WHAT-IF NUMBERS, NOT LIVE (N VALIDATOR MODIFIED)"
  — is the dashboard's single best irreversibility microcopy. The
  persona's #1 fear (real vs pretend money) is resolved on first
  glance. Worth porting the pattern elsewhere.
- **Pattern:** the dashboard is *learnable* (manual closes most gaps)
  but not yet *legible* (chrome doesn't link the manual from where
  confusion arises). Cleanup is mostly a labelling and discoverability
  project, not a redesign.

Full summary: `docs/ux-13yo/summary.md`. Per-screen reports under

## What the pipeline taught us about itself

The dogfood surfaced operational flaws in the brand-new skills,
within hours of writing them. Sent both skills back to oracle for a
second adversarial pass — critique at
`docs/ux-13yo/oracle-skills-critique.md`. Verdict: pipeline is
directionally useful but not yet a reliable skill-minting process.
Specific failure modes:

- 4 parallel walkthrough subagents hit usage caps before writing
  reports. Screenshots survived; markdown didn't. Salvage loop (2
  lighter writer-only subagents) recovered the data.
- The "write the report at the end" pattern is fragile under
  interruption. Should be "create file first, append per step".
- `create`'s dogfood step is one line — exactly the kind of guard rail
  that gets skimmed past.
- Subagents may be sandbox-blocked from writing `~/.claude/skills/`;
  main thread must own the final SKILL.md write.
- `blocker / friction` is still severity ranking, just coarser —
  some reports smuggled `small / major / minor` back in despite the
  NEVER rule.

Proposed updates for both skills tracked in
`docs/ux-13yo/skill-updates.md`.

## v2 of both skills (later same day)

Sent the v1 skills back to oracle for adversarial second pass. Critique
(`docs/ux-13yo/oracle-skills-critique.md`) said: directionally useful
but "not fit for purpose as-is" — operational hardening between "good
idea" and "reusable skill" is the gap.

Applied changes:

- `create` renamed → `scavenge`. The metaphor is honest: we don't
  invent, we scavenge known best practices (NN/g, Norman, Krug,
  GOV.UK, ...) and assemble what survives oracle.
- `scavenge` broadened beyond SKILL.md — now produces any of seven
  durable shapes: skill, agent, guideline, runbook, checklist, review
  template, spec. Step 1 picks the shape by REUSE + INVOCATION
  pattern; step 5 branches.
- `scavenge` got **critique checkpoints at every step**, not just at
  step 3. Oracle threads through (light sanity at step 1, deep
  adversarial at step 3, post-write check at step 5, dogfood-as-
  critique at step 6).
- `eye-13y` renamed → `eye-13yo` (matches `ux-13yo` docs dir).
- `eye-13yo`: `blocker / friction` tags removed (they were still
  severity ranking and dogfood proved synonyms leaked back in).
  Replaced with `cite / inference` — **evidence type, not severity**.
- `eye-13yo`: scaffold-report-file-first promoted to setup; subagents
  append per step, never write only at the end (the failure mode
  that lost work today).
- `eye-13yo`: cold findings and post-manual findings now hard-separated
  as distinct headings — retroactive reclassification destroyed
  evidence in v1.
- `eye-13yo`: finance overlay is automatic on any money-bearing screen
  (trigger list), not "if you notice". Six fixed lines, fixed order.
- Both skills: default 1–2 subagents, not ≤4. Explicit agent-browser
  invocation via Skill tool. Salvage rule for interrupted subagents.

Proposals trail: `docs/ux-13yo/skill-updates.md` (marked applied at
top).
