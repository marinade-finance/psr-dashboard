# Plan 01-04: Mobile Responsiveness + Polish

## Objective
Make the terminal design work well on mobile and polish overall alignment.

## Tasks

### Task 1: Mobile terminal output
- Wrap all pre-formatted terminal output in `overflow-x-auto` containers
- On mobile (<768px), show a simplified view:
  - Stats: stack vertically, one per line `:: LABEL .... value`
  - Tables: allow horizontal scroll with a subtle `→ scroll` indicator
  - Nav: collapse to `> PSR` with hamburger `[≡]` showing links
- Add `-webkit-overflow-scrolling: touch` for smooth mobile scroll

### Task 2: SAM table alignment polish
- Ensure all columns align perfectly (verify pad widths match content)
- Make sure long validator names truncate cleanly
- Verify ASCII bars render consistently

### Task 3: Clean up unused components
- Remove or simplify ui/table.tsx (no longer used by main tables)
- Ensure card.tsx, badge.tsx work as terminal fallbacks for any remaining usage
- Remove unused CSS (dark mode remnants, etc.)

### Task 4: Loading states
- Verify Loader component has the cycling `&#^#*!(@$%` characters
- Skeleton components show `░` blocks that pulse
- Error states show terminal-style: `!! ERROR: message`

### Success Criteria
- [ ] Mobile: horizontal scroll works, stats stack
- [ ] Nav: responsive hamburger on mobile
- [ ] Columns align on all three tables
- [ ] No dead CSS/components
- [ ] Builds without errors
